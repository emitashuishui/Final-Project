Final Project Report: Property Rental Management System
1. Summary
The Property Rental Management System is a relational database designed to streamline the management of residential real estate. It automates the tracking of property owners, rental units, tenant information, lease agreements, and payment history. The system ensures data integrity and reduces redundancy through a normalized schema.
2. Introduction
In the real estate industry, managing multiple properties manually often leads to errors in lease tracking and missed payments. This project provides a robust backend solution using Oracle SQL. The goal is to provide a single source of truth for landlords and property managers to monitor their portfolio performance and tenant compliance.
3. Data Modeling
The database was designed using an Entity-Relationship (ER) Diagram to map out the logical structure before physical implementation.
•	Entities: Owners, Tenants, Properties, Leases, Payments.
•	Cardinality:
o	One Owner can own multiple Properties (1:N).
o	One Property can have multiple Leases over its lifetime (1:N).
o	One Lease can have multiple Payments (1:N).
4. Design Decisions
To meet the project requirements, the following architectural choices were made:
•	Normalization (3NF): I separated Owners and Tenants into their own tables. This ensures that if a tenant’s phone number changes, it only needs to be updated in one place, preventing "update anomalies."
•	Referential Integrity: I used FOREIGN KEY constraints with CASCADE CONSTRAINTS. This prevents a property from being assigned to an owner that doesn't exist.
•	Data Types: I chose NUMBER(10, 2) for rent and payments to ensure financial precision, and VARCHAR2 for text to optimize storage in the Oracle environment.
•	Automation: Used GENERATED BY DEFAULT AS IDENTITY for all Primary Keys to allow the database to handle ID generation automatically.
